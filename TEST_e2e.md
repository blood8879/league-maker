### 실제 실패 기록

#### 실패 #1
- **테스트 항목**: 2.2.3 - 로그인 버튼 표시
- **실패 날짜**: 2025-11-30
- **담당자**: Claude
- **우선순위**: P0
- **실패 원인**:
  - [x] UI 요소 중복 / Strict mode violation
  - [ ] 타임아웃 (예상 시간 초과)
  - [ ] 예상치 못한 에러 메시지
  - [ ] 기능 미구현
  - [ ] API 응답 오류
  - [ ] Mock 데이터 문제
  - [ ] 네트워크/환경 문제
  - [ ] 기타
- **에러 메시지**:
  ```
  Error: strict mode violation: getByRole('button', { name: '로그인' }) resolved to 2 elements:
  1) <button>로그인</button> aka getByRole('link', { name: '로그인' }).getByRole('button')
  2) <button type="submit">로그인</button> aka getByRole('main').getByRole('button', { name: '로그인' })
  ```
- **재현 단계**:
  1. `/login` 페이지 접속
  2. `getByRole('button', { name: '로그인' })` 선택자로 버튼 찾기
  3. 헤더와 본문에 각각 "로그인" 버튼이 존재하여 strict mode violation 발생
- **예상 동작**: 로그인 폼의 제출 버튼만 선택되어야 함
- **실제 동작**: 헤더 네비게이션의 "로그인" 버튼과 폼의 "로그인" 버튼 2개가 모두 선택됨
- **해결 방법**:
  - 테스트 코드에서 더 구체적인 선택자 사용: `page.getByRole('main').getByRole('button', { name: '로그인' })`
  - 또는 헤더 버튼에 다른 텍스트 사용 (예: "로그인하기")
